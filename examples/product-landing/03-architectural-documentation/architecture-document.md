## 1. Purpose, Audiences And Uses

Engineering and future agents use this document to preserve the static boundary, quality goals, and provider constraints; product/design uses it to understand required evidence.

## 2. System Context And Scope

The system is one static public page delivered by the existing CDN. Signup and personal data belong to the external provider; no runtime backend or database is in scope.

## 3. Architectural Drivers And Scenarios

DRV-001 requires P75 LCP at or below 2.5 seconds; DRV-002 requires the content and CTA to meet the defined WCAG 2.2 AA review scope.

## 7. Interfaces And Runtime Behavior

Browsers request versioned assets over HTTPS and follow a normal link to the approved signup provider. Analytics loads only under the approved consent policy.

## 8. Quality, Security, Data And Operations

Optimize images/fonts, use semantic HTML and visible focus, keep personal data external, monitor availability/performance, and retain the previous static release for rollback.

## 10. Risks, Assumptions And Technical Debt

Brand asset weight and contrast remain owned by product/design; an editor-driven content requirement would trigger ADR review.

## 11. Governance And Agent Guidance

Agents must read ADR-001, preserve the no-backend boundary, run CHECK-001/CHECK-002, and reopen phase 2 before adding a runtime service or direct personal-data collection.
