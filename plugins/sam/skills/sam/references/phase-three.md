# Phase 3 Contract - Architectural Documentation

## Select Views From Questions

Identify audiences, concerns, questions, and intended uses first. Select the minimum useful views afterward. C4, module, runtime, deployment, data, security, class, or sequence views are options, not requirements.

For each selected view record:

- audience and question;
- elements, responsibilities, boundaries, and relations;
- diagram or table;
- notation and legend;
- rationale, assumptions, and limitations.

If no optional view is useful, record the architect-approved rationale instead of adding a placeholder diagram.

## Cross-View Information

Consolidate system context/scope, drivers/scenarios, ADRs, interfaces and behavior, security/data/operations concerns, risks, technical debt, evidence, governance, and agent reading guidance. Use N/A with rationale for inapplicable concerns.

## Approval Readiness

- Views agree with approved ADRs.
- Primary drivers trace to ADRs or Accepted Risk and CHECK definitions.
- Applicable context, security, data, operations, migration, recovery, and evolution concerns are covered or accepted.
- Agents are told which artifacts are authoritative and when to reopen SAM.
- Source hashes, risks, and review triggers are current.
